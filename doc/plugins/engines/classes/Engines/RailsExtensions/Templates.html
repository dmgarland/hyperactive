As well as providing code overloading for controllers and helpers (see Engines::RailsExtensions::Dependencies), the engines plugin also allows developers to selectively override views and partials within their application.

An example

This is achieved in much the same way as controller overriding. Our plugin contains a view to be rendered at the URL /test/hello, in our_plugin/app/views/test/hello.rhtml: 

   <div class="greeting">Hi, <%= @dude.name %>, what's up?</div>

If in a particular application we‘re not happy with this message, we can override it by replacing the partial in our own application - RAILS_ROOT/app/views/test/hello.rhtml: 

    <div class="greeting custom_class">Wassup <%= @dude.name %>.
                                       Waaaaassaaaaaaaaup</div>

This view will then be rendered in favour of that in the plugin.
